# Changelog

Feature-only history for the `vsIronBar-Studio` Version 1 series.
Bug-fix-only releases are intentionally omitted from this summary.

---

## [1.2.0] - 2026-03-29

### Features

- Lazy tab loading for faster startup
- Sticky header with always-visible actions and live preview

---

## [1.1.0] - 2026-03-29

### Features

- Dock support as a full second bar configuration
- WebKit2 live preview with real CSS rendering
- Combined bar + dock preview
- Config auto-detection at startup
- Open dialog for choosing config and stylesheet together
- Permanent `backups/original/` first-contact snapshot
- Tab icons across the editor
- `vsHub` tab for discovering and launching related tools
- JSONC support for comments and trailing commas
- Named module instance support
- Preservation of unknown modules and config blocks
- Layout validation alert for unrecognized entries
- Per-module documentation links

---

## [1.0.1] - 2026-03-21

### Features

- User Commands for custom launcher modules
- Built-in `custom/settings` module
- Automatic backups on save operations
- Bar-style detection when reopening existing stylesheets
- Saved scripts marked executable automatically
- Bundled `weather.sh` fallback script
- Ironbar process tracking on restart

---

## [1.0.0] - 2026-03-21

### Features

- Initial single-file Python 3 + GTK3 release
- Bar editor for core layout and placement settings
- Layout editor for left / center / right module zones
- Modules tab with editable Ironbar module forms
- Styling tab with CSS color token editing
- Templates tab with ready-made presets
- Scripts tab with weather integration
- Built-in palette library for multiple themes
